# Accessibility Policy
## 1. キーボード完走（必須）
- メニュー/主要ボタン/ダイアログ/キャンバス操作がキーボードのみで完結する
- Esc: 現在操作のキャンセル → 安全な状態（Select/Idle）へ戻る
- Enter/Space: ボタン/トグルの実行
- Tab/Shift+Tab: フォーカス移動（論理順）

## 2. フォーカス
- フォーカスリングは必ず可視（ユーザーが「今どこか」分かる）
- モーダルはフォーカストラップ（外へ出ない）
- 初期フォーカスは最初に操作すべき要素へ

## 3. 読み上げ（アクセシブル名/ロール）
- 主要UI要素はアクセシブル名を持つ（i18n key で指定）
- ロール（Button/TextField/List 等）をUIフレームワークの機構に従い設定する

## 4. ショートカット
- すべてテーブル定義で管理し、衝突を起動時/テストで検知する
- OS差: Win/Linux=Ctrl、macOS=Cmd を吸収する
- モード依存を明記し、Sprint20のモード遷移と衝突しない
