# i18n Policy
## 0. 直書き禁止
UIに表示する文字列は必ず i18n key を参照する。例外は「内部識別子」「開発専用ログ」のみ。

## 1. 参照方法
- key: UI.<AREA>.<COMPONENT>.<ACTION>
- UI層は i18n::Catalog を介して文字列を取得する（キー直書きは可、値直書きは禁止）

## 2. プレースホルダ
- 形式: {name}
- 未提供引数は開発で気付けるよう [[MISSING:name]] を表示する（本番でも安全側）

## 3. フォールバック
実行時フォールバックは en -> ja を許可してよい。ただし主要導線の欠落はCIでエラー。

## 4. 入力耐性（limits）
- 翻訳値の長さは上限を持つ（schemaで maxLength を規定）
- 危険な制御文字は置換/除去する（\r, \u202E 等）
