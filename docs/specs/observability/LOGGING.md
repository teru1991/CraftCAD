# LOGGING

最終更新: 2026-02-28  
ステータス: 草案

## 1. ログ方針

- アプリログは標準出力（stdout）とログファイルの双方に出力する。
- ログファイルは OS 標準のアプリデータ配下 `logs/` に配置する。
- 初期実装ではローテーションは未対応（将来タスク）。
- `.diycad`（プロジェクト本体）とは分離して管理する。

## 2. PII（個人情報）を入れないルール

- ログにユーザーの個人情報（氏名、メール、住所、電話番号など）を書き込まない。
- ファイルパスや入力値を記録する際は必要最小限にとどめ、識別情報の露出を避ける。
- 例外/エラー時も機密情報をそのまま出力しない。

## 3. ユーザー削除可能性

- ログはユーザーが削除可能であることを前提に設計する。
- 将来タスクとして、設定画面または CLI からログ削除機能を提供する。
