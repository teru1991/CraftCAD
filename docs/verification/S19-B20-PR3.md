## SSOT整合（policy/shortcuts/required_shortcuts と実装テーブル）
- `docs/specs/a11y/policy.md` でキーボード完走・フォーカス・読み上げ・ショートカット衝突検知の契約を固定。
- `docs/specs/a11y/shortcuts.md` は人間向け一覧、`docs/specs/a11y/required_shortcuts.json` はCIで検証する機械可読正本に分離。
- `apps/desktop/ui/a11y/shortcuts.rs` はテーブル定義（`SHORTCUTS`）として実装し、`check_shortcuts_table` / `format_conflicts` で衝突/重複を検出する責務を明示。
- `core/serialize/tests/spec_ssot_lint.rs` には SSOT（`required_shortcuts.json`）の構造・重複・空要素チェックを追加し、契約逸脱をCIで fail するようにした。

## 衝突検知の仕組み
- `check_shortcuts_table()`:
  - `scope -> chord -> ids` のマップを構築し、同一 scope で同一 chord に複数 id が割り当たると conflict として収集。
  - shortcut id の重複も別途検出。
- `format_conflicts()`:
  - conflict/重複がある場合に読みやすい multi-line string を返し、起動時エラー化・テスト失敗メッセージに利用可能。
- `apps/desktop/ui/a11y/tests/shortcuts_table.rs`:
  - conflict が 0 であることを assert（fail-fast）。
