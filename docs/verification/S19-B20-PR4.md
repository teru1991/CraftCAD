# S19-B20-PR4 Verification

## 主要画面でのキーボード完走（実装範囲）
- PR4では UI フレームワーク依存を避けるため、`apps/desktop/ui/**` に抽象ハンドラ/状態/フォーカス管理を実装した。
- 実装済みのロジック到達範囲:
  - ShortcutScope + KeyChord 正規化から Action 解決（Open/Save/Undo/Redo/Esc/Search/Help/Pan/Zoom/Tool/Nest/Export）
  - Esc は `AppState::apply_action(AppAction::Cancel)` で `SafeMode::Select` + `ToolMode::Select` に戻す
  - Tab/Shift+Tab は `FocusManager` で順序遷移、モーダル設定時はトラップ
  - 設定モデル（Ja/En, Metric/Imperial, scale, font size）保持と i18n runtime 再生成
- 既存 Qt 実画面への全面接続は PR5 以降（このPRは SSOT 契約の実行ロジックを先に固定）。

  - SSOT テーブル定義、衝突検知、ID重複検知、1キー系判定

  - `(scope, chord) -> AppAction` 解決、テキスト入力中ブロック判定
- `apps/desktop/ui/a11y/focus_manager.rs`
  - FocusChain 管理、Tab/Shift+Tab 遷移、trap 制御
- `apps/desktop/ui/a11y/focus_order.rs`
  - project/support/settings の標準フォーカス順を chain 化
- `apps/desktop/ui/a11y/accessibility.rs`

- `apps/desktop/ui/screens/{project,support_dialog,settings}.rs`
  - 画面単位で focus manager + a11y props を利用する最小接続
